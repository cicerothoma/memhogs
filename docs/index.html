The problem

Modern apps are not single processes. A browser or an Electron app (VS Code, Slack, Cursor) spreads its memory across dozens of helper processes with names like Browser Helper (Renderer) or Code Helper (Plugin). Tools like ps, top, and Activity Monitor list each one separately, so the app that is using 7 GiB of your RAM never appears as a 7 GiB entry anywhere.

It gets worse. Some binaries hide their identity entirely (Warp's executable is named stable), the obvious ps incantation sorts by CPU rather than memory, and adding up RSS values double-counts memory shared between processes, inflating totals by gigabytes.

memhogs fixes all of that in one command.

What it does

Groups by app

Walks the process tree, so every helper, renderer, and language server rolls up into the application that spawned it. One app, one row, one total.

Real names

Resolves processes to the .app bundle on macOS and the systemd unit on Linux. Warp shows up as Warp, not stable.

Honest numbers

Uses memory footprint on macOS (the Activity Monitor metric) and PSS on Linux, so shared memory is never counted twice. Plain RSS is one flag away.

Respects your terminal

Jobs you launch from a shell stay separate. A 4 GiB training run started in a terminal tab shows up as python3, not as your terminal app.

Scriptable

JSON output with raw byte counts for dashboards and alerts. Filter by name, cap the list, or watch it live.

Small and fast

A single static binary for macOS and Linux, amd64 and arm64. No runtime, no configuration, no dependencies.

When you'd reach for it

  • Your laptop fan is spinning and you want the culprit ranked, named, and totalled in one command.
  • You are on a remote Linux server where Activity Monitor does not exist and top shows you five hundred anonymous rows.
  • You want to know what quitting your browser would actually give back, including all its renderers.
  • You are watching a suspected memory leak grow with memhogs --watch myapp.
  • You feed memhogs --json into a script that alerts when any one app crosses a threshold.

Install

Homebrew (macOS and Linux)

brew install cicerothoma/tap/memhogs

Debian / Ubuntu

# download the .deb from the latest release, then
sudo dpkg -i memhogs_*.deb

Fedora / RHEL / Alpine

sudo rpm -i memhogs-*.rpm          # Fedora, RHEL
sudo apk add --allow-untrusted memhogs_*.apk   # Alpine

Go

go install github.com/cicerothoma/memhogs@latest

Prebuilt tarballs for every platform are on the releases page.

Commands at a glance

memhogs                 # groups sorted by memory, top 5 members each
memhogs --tree          # expand every member process
memhogs --compact       # one row per group
memhogs --flat          # individual processes, no grouping
memhogs --top 10        # only the 10 largest
memhogs --json          # machine-readable output
memhogs --watch         # live view, refreshes every 2s
memhogs --rss           # ps/top-comparable numbers
memhogs slack           # only groups matching "slack"

The README shows example output for each one.
